Club kits

Nav: Admin → Season Break → Club kits → Download latest kits

Set image paths or URLs for each club's home, away, and third kits. Owners see these on Club Details. Leave a field blank to use the default path images/clubs_kits/{SHORT}_home.png (and _away / _third), or use Sync from Colours of Football below to pull the latest kits automatically. Graphics courtesy of colours-of-football.com.

One-time setup (required)

GitHub deploy alone does not install the edge function. Until this is done, Download latest kits will show “Failed to send a request to the Edge Function”.

  1. Supabase Dashboard → Edge Functions → Create function
  2. Name exactly: club-kits-cof-sync
  3. Paste all of supabase/functions/club-kits-cof-sync/index.ts from this repo
  4. Deploy → Settings → turn OFF “Enforce JWT verification” (same as GPDB scrape)
  5. Optional: Storage → create public bucket club-kits (for downloaded PNGs)
  6. Run supabase/sql/patches/club_kits.sql if not already applied

See scripts/README_club_kits_cof.md. No Supabase CLI needed — Dashboard paste is enough. Or run python scripts/fetch_club_kits.py locally to fill images/clubs_kits/ without the edge function.

Download latest kits

Pulls the current-season home, away, and third kits from colours-of-football.com by reading on-page headers such as home kit 2025-2026 / 25-26, then matching kit images for that season only. Requires edge function club-kits-cof-sync (deploy index.ts + club_kits_cof.js).

Untick storage if you only want hotlink URLs saved to the database. For static files in the repo, run python scripts/fetch_club_kits.py locally.
