1. Scrape PESDB → staging

PESDB allows about 2 list pages per session (~32 players each, ~2 min/page with details). This scraper works in 2-page batches: full list + max rating + playing style per player, then a short cooldown before the next batch (same logic as the local Python script). Progress is saved to the database after each page — you can refresh this tab and the scrape will auto-continue. Closing the browser entirely pauses until you open this page again. Test with pages 1–2 first. CSV upload still available as fallback.

Or upload an existing scrape CSV (e.g. from scripts/pesdb_scrape.py).

2. Preview sync

Dry run — no changes to Players. Check the summary counts first; the table defaults to Updates (existing GPDB) so you can compare ratings/MV. New PESDB cards are listed separately (often thousands — not shown in the first 250 by default).

3. Apply sync

Updates matched players; marks missing cards as legacy; inserts new cards as free agents.

Legacy cards (off PESDB)

Owners may keep these at their club. Not sellable. Renew 1 season at a time. Restore manually if a card returns before the next full sync.
