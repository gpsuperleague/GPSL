Discord News Feed

Channels

#gpsl-news — transfers (filtered), managers, owners, cups, titles, etc.
Transfer filter: age ≤20 need rating ≥70; age ≥21 need rating ≥76.
#gpsl-results — confirmed match results only (month · competition · score).
#gpsl-natter — owner Natter posts (text + image).

  1. In Discord: create channels #gpsl-results and #gpsl-natter if needed.
  2. For each: Channel settings → Integrations → Webhooks → New Webhook → Copy URL (create the webhook inside that channel).
  3. Supabase → Edge Functions → Secrets (project-wide, not inside a function):
    • DISCORD_WEBHOOK_URL = #gpsl-news
    • DISCORD_RESULTS_WEBHOOK_URL = #gpsl-results
    • DISCORD_NATTER_WEBHOOK_URL = #gpsl-natter
  4. SQL: gpsl_discord_results_channel_and_transfer_filter.sql and gpsl_discord_natter_posts.sql.
  5. Redeploy: supabase functions deploy discord-sky-feed.
  6. Use Check routing then the channel test buttons.

Auto-post (no button)

Events are queued in the database. With auto-post enabled, Postgres calls the edge function as soon as something is queued (plus a 2-minute retry cron).

Manual Push uses your admin login (works even when auto doesn’t).
Auto-post must use the project service_role key (Supabase → Project Settings → API → service_role), not a random string — the gateway rejects custom keys.
Also run gpsl_discord_feed_auto_flush_fix.sql and enable pg_net.
