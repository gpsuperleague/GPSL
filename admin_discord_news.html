Auto-post (no button)

Events are queued in the database. With auto-post enabled, Postgres calls the edge function as soon as something is queued (plus a 2-minute retry cron).

Manual Push uses your admin login (works even when auto doesn’t).
Auto-post must use the project service_role key (Supabase → Project Settings → API → service_role), not a random string — the gateway rejects custom keys.
Also run gpsl_discord_feed_auto_flush_fix.sql and enable pg_net.
